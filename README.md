# SomnoScope

Сервис для предварительной диагностики нарушений дыхания сна по данным ЭЭГ.  
Проект разработан в рамках хакатона **«Цифровой прорыв»**.


## О проекте

SomnoScope анализирует полисомнографические записи в формате `.REC`, выполняет предобработку сигнала и помогает выявлять признаки нарушений дыхания сна. Решение сочетает:

- `Chrononet` на `PyTorch` для бинарной классификации наличия нарушения;
- `CatBoost` для оценки индекса `HI` при положительном результате;
- `Streamlit` для быстрого запуска демонстрационного веб-интерфейса;
- `MNE` для чтения и обработки ЭЭГ-сигналов.

Демо-версия: http://team.927.webtm.ru/

## Возможности

- загрузка `.REC`-файлов через веб-интерфейс;
- автоматическая нормализация заголовка записи и чтение сигнала через `MNE`;
- анализ фиксированных 150-секундных эпох по ключевым ЭЭГ-каналам;
- визуализация спектра сигнала и репрезентативных фрагментов записи;
- оценка индекса `HI` для записей с выявленными признаками НДС.

## Работа с данными

Приложение ожидает записи в формате `.REC`. Во время загрузки файл временно конвертируется в `.EDF`, после чего запускается предобработка и инференс.

Поддерживаемые каналы:

- `Fp1-M2`
- `C3-M2`
- `O1-M2`
- `Fp2-M1`
- `C4-M1`
- `O2-M1`

## Технический пайплайн

Инференс реализован в [`src/sp_dfo/inference.py`](src/sp_dfo/inference.py) и построен как двухступенчатый пайплайн:

1. Бинарный классификатор `Chrononet` определяет, есть ли у записи признаки нарушения дыхания сна.
2. Если запись отнесена к положительному классу, отдельный регрессор `CatBoost` оценивает индекс `HI` по эмбеддингам, извлеченным из классификатора.

### Предобработка сигнала

- при загрузке файла нормализуется проблемный формат времени в заголовке EDF-совместимой записи, из-за которого `MNE` может некорректно читать `.REC`;
- сигнал приводится к общей референсной схеме через `set_eeg_reference()`;
- применяется полосовая фильтрация `1-45 Hz`, чтобы отсечь медленные дрейфы и высокочастотный шум перед инференсом;
- запись режется на неперекрывающиеся эпохи длительностью `150` секунд;
- из полного набора каналов выбираются шесть целевых ЭЭГ-каналов, а отсутствующие каналы заменяются нулевыми дорожками, чтобы сохранить фиксированную входную размерность модели.

### Классификационная модель

`Chrononet` определена в [`src/sp_dfo/model.py`](src/sp_dfo/model.py) и использует каскад из нескольких уровней:

- три последовательных сверточных блока `Conv1d` с мульти-масштабными ядрами `2`, `4` и `8`, которые параллельно извлекают локальные паттерны разной длительности;
- стек `GRU`, который агрегирует временной контекст после сверточного извлечения признаков;
- линейное сжатие временной оси и финальный рекуррентный блок для формирования компактного представления записи;
- выходной классификатор на `2` класса и скрытое представление, используемое как embedding для второго этапа пайплайна.

### Агрегация по эпохам

- модель делает предсказание отдельно для каждой 150-секундной эпохи;
- итоговый класс записи выбирается большинством голосов по всем эпохам;
- для визуализации интерфейс выбирает наиболее уверенную эпоху по максимальной softmax-уверенности;
- при положительном решении эмбеддинги всех эпох передаются в `CatBoost`, а итоговый `HI` усредняется по предсказаниям регрессора.

## Модели

В репозиторий уже включены артефакты инференса:

- `models/best_model.pth` — веса классификатора `Chrononet`;
- `models/cb_model.cbm` — регрессионная модель `CatBoost` для оценки `HI`.

Ноутбук для экспериментов и обучения находится в `notebooks/train.ipynb`.

Текущая реализация ориентирована именно на inference-сценарий: веса загружаются лениво и кэшируются в памяти процесса, чтобы не инициализировать модели повторно на каждый новый загруженный файл в Streamlit-сессии.

## Ограничения

- проект предназначен для демонстрации и прототипирования;
- результат модели не должен использоваться как единственный источник медицинского решения;
- для корректной оценки качества на новых данных важно соблюдать исходную структуру каналов и формат записи.
